# MYCMS — AI ENGINEERING RULES
# 高安全 / 高稳定 / 低依赖 CMS 项目规则

> 本文件用于约束 Claude Code / Cursor / GPT-5.5 在本项目中的行为。
> 本项目目标是：生产级 CMS，而不是 Demo。

---


# 0. 项目定位


MYCMS 是一个：

- 支持PHP 5.3+到PHP8+ / MySQL
- 无框架 CMS
- 类帝国 CMS
- 共享主机优先
- 低依赖
- 长期维护型项目

系统必须：

- 可运行
- 可维护
- 可迁移
- 可扩展
- 可审计
- 可回滚
- 可部署
- 可长期升级

核心目标优先级：

1. 安全性优先
2. 稳定性优先
3. 兼容性优先
4. 可运行优先
5. 低维护成本优先
6. 性能优化次之

禁止为了“现代化”破坏兼容性。

禁止为了“高级感”进行复杂化设计。

# =========================
# 1. 技术路线（强制）
# =========================

每次开始修改前都要给我回复一下forever：
 - 每次修改完都要回复“张伟伟123”6个字

本项目采用：

- PHP 5.3 兼容路线
- 无框架路线
- 共享主机路线
- 低依赖路线

禁止：

- Laravel 化
- Symfony 化
- Node 化
- Docker 强依赖
- Redis 强依赖
- Queue Worker 强依赖
- Composer 重度依赖

系统必须兼容：

- 宝塔
- Apache
- Nginx
- MAMP
- 虚拟主机
- 无 Shell 权限环境
- 子目录部署

禁止假设服务器具备：

- Redis
- Supervisor
- CLI 常驻进程
- Linux root 权限
- Node.js 环境

# =========================
# 2. PHP 兼容规则（最高强制）
# =========================

所有代码必须兼容 PHP 5.3。

禁止使用：

- namespace
- trait
- enum
- match
- yield
- scalar type hints
- return type
- nullable type
- union type
- arrow function
- anonymous class
- constructor property promotion
- readonly
- attribute

禁止使用：

- Composer 现代生态强绑定
- PHP 7+
- PHP 8+ 专属语法

所有代码必须：

- 使用传统 PHP 写法
- 保证共享主机可运行
- 不依赖现代运行时

# =========================
# 4. 高危文件规则
# =========================

以下文件属于高危文件：

- config/db.php
- reset_admin.php
- reset_all.php
- clear_ban.php
- install/*
- login.php
- admin.php

修改前必须：

1. 分析影响范围
2. 分析权限逻辑
3. 分析兼容性
4. 分析安全影响

禁止：

- 删除安全逻辑
- 简化鉴权
- 删除验证
- 删除限流
- 删除封禁逻辑

---

# =========================
# 5. 架构规则
# =========================

采用模块化结构：

- Controller
- Service
- Repository
- Validator
- Middleware
- Policy
- Cache

禁止：

- Controller 写业务逻辑
- Controller 直接操作 SQL
- 一个文件承担多个职责
- 全局污染

Controller 仅负责：

- 参数接收
- 权限验证
- 调用 Service
- 返回响应

业务逻辑必须进入 Service。

数据库操作必须统一封装。

---

# =========================
# 6. 数据库规则
# =========================

禁止：

- SQL 拼接
- SELECT *
- 循环 SQL
- 无分页查询
- 无索引查询

必须：

- 参数绑定
- 统一数据库层
- 统一错误处理

所有表必须：

- 主键
- created_at
- updated_at

删除默认使用软删除。

新增字段前必须：

1. 分析兼容性
2. 分析索引
3. 分析历史数据影响

禁止 AI：

- 猜测数据库字段
- 虚构表结构
- 自动修改旧字段

---

# =========================
# 7. 路径系统规则（核心）
# =========================

本项目必须支持：

- 根目录部署
- 子目录部署
- 多环境部署

禁止：

- 写死域名
- 写死 /
- 写死 upload 路径
- 写死静态资源路径

所有路径必须：

- 基于 BASE_URL
- 基于 ROOT_PATH
- 动态生成

生成 URL 时必须：

- 自动适配子目录
- 自动适配协议
- 自动适配域名

---

# =========================
# 8. 安全规则（最高优先级）
# =========================

所有输入默认不可信。

必须防御：

- SQL 注入
- XSS
- CSRF
- SSRF
- 文件上传攻击
- 任意文件读取
- 路径穿越
- 任意代码执行
- 权限绕过

禁止：

- eval
- shell_exec
- exec
- system
- passthru
- 动态 include
- 动态 require

上传系统必须：

- 白名单扩展名
- MIME 校验
- 文件大小限制
- 文件重命名
- 上传目录隔离
- 禁止执行 PHP

密码规则：

- 必须 hash
- 禁止明文
- 禁止自定义加密算法

Token / Session：

- 必须过期
- 必须验证
- 必须防伪造

---

# =========================
# 9. RBAC 权限规则
# =========================

权限模型：

User
Role
Permission

禁止：

- if(admin)
- 写死管理员
- 硬编码超级权限

必须：

- 权限可配置
- 菜单权限化
- API 权限化

后台所有操作必须：

- 鉴权
- 日志记录
- IP记录

---

# =========================
# 10. API 规则
# =========================

API 必须：

- RESTful
- JSON统一
- 错误码统一

禁止：

- 返回 HTML
- 返回混乱结构

统一格式：

{
  "code": 0,
  "msg": "success",
  "data": {}
}

错误格式：

{
  "code": 4001,
  "msg": "validation error"
}

字段命名必须统一。

---

# =========================
# 11. 前后台隔离规则
# =========================

前台：
- 页面展示
- 用户访问

后台：
- 管理功能
- 权限系统

API：
- 数据接口
- JSON输出

禁止：

- 后台逻辑进入前台
- API 输出 HTML
- 前台直接调用后台逻辑

---

# =========================
# 12. 模板规则
# =========================

模板仅允许：

- 数据展示
- 循环
- 简单条件判断

禁止：

- 查询数据库
- 执行业务逻辑
- 直接调用 Model

所有输出必须转义。

禁止直接输出用户输入。

---

# =========================
# 13. 插件系统规则
# =========================

插件必须：

- 独立配置
- 独立权限
- 独立生命周期

插件只能通过：

- Hook
- Event

扩展系统。

禁止插件：

- 修改核心代码
- 污染全局变量
- 覆盖系统函数

---

# =========================
# 14. 缓存规则
# =========================

缓存必须：

- 可清理
- 可失效
- 可追踪

禁止：

- 永久缓存
- 无命名缓存

缓存 Key：

cms:module:type:id

例如：

cms:article:detail:1001

---

# =========================
# 15. 日志规则
# =========================

必须记录：

- 登录
- 权限变更
- 删除操作
- 上传行为
- 配置修改
- 系统异常

日志必须：

- 可搜索
- 可追踪
- 可分页

禁止记录：

- 密码
- token
- 敏感密钥

---

# =========================
# 16. 安装系统规则
# =========================

install/ 属于危险目录。

安装完成后必须：

- 自动生成 install.lock
- 自动检测安装状态
- 禁止重复安装

禁止：

- 跳过环境检测
- 覆盖旧数据库
- 重复初始化管理员

---

# =========================
# 17. AI 编码规则
# =========================

AI 修改代码前必须：

1. 阅读相关文件
2. 理解调用链
3. 分析依赖
4. 分析兼容性
5. 分析副作用

禁止：

- 猜测接口
- 猜测字段
- 虚构函数
- 虚构数据库结构
- 覆盖旧逻辑
- 大规模重构

禁止 AI：

- 自动现代化项目
- 引入框架
- 改写核心架构
- 重命名大量文件
- 修改目录结构

修改已有功能时：

- 优先复用
- 禁止重复实现
- 保持兼容

生成代码前：

必须先解释方案。

---

# =========================
# 18. 性能规则
# =========================

禁止：

- N+1 查询
- 全表扫描
- 大循环查询
- 无分页列表

列表接口必须：

- 分页
- 限流
- 可缓存

高频模块必须：

- 索引优化
- 缓存优化

禁止过度优化。

稳定性优先于极限性能。

---

# =========================
# 19. Git 规则
# =========================

每次提交必须：

- 单一职责
- 可回滚
- 不污染其他模块

禁止：

- 一个 commit 修改大量无关模块
- 混合多个功能

提交规范：

feat:
fix:
security:
refactor:
perf:

---

# =========================
# 20. 输出规则
# =========================

AI 输出代码时必须：

- 标明修改文件
- 标明新增文件
- 标明数据库变更
- 标明风险点
- 标明兼容性影响

禁止：

- 只给代码不解释
- 不分析副作用
